# FCL启动器崩溃排查与修复

在使用FCL启动器运行Minecraft时，偶尔可能会遇到游戏崩溃的情况。本指南将帮助您诊断和修复常见的崩溃问题。

## 常见崩溃原因

<div style={{marginBottom: '20px'}}>
  <h3 style={{color: '#d73a49', marginBottom: '10px'}}>内存不足</h3>
  <p>
    Minecraft Java版需要较多的内存资源，特别是在安装了大量模组或高清材质包时。
  </p>
  
  <h3 style={{color: '#d73a49', marginBottom: '10px'}}>模组冲突</h3>
  <p>
    某些模组可能与其他模组或特定版本的Minecraft不兼容。
  </p>
  
  <h3 style={{color: '#d73a49', marginBottom: '10px'}}>Java版本问题</h3>
  <p>
    使用不兼容的Java版本可能导致游戏崩溃。
  </p>
</div>

## 读取崩溃日志

崩溃日志是诊断问题的关键。FCL启动器会自动生成崩溃报告，您可以按照以下步骤找到它：

<ol style={{backgroundColor: '#f6f8fa', padding: '15px 15px 15px 40px', borderRadius: '6px'}}>
  <li>打开FCL启动器</li>
  <li>点击右上角的菜单图标</li>
  <li>选择"设置"</li>
  <li>滚动到"诊断"部分</li>
  <li>点击"查看崩溃报告"</li>
</ol>

## 常见崩溃解决方案

### 1. 增加内存分配

如果崩溃是由内存不足引起的，可以通过以下步骤增加分配给Minecraft的内存：

<div style={{display: 'flex', flexDirection: 'column', gap: '10px', margin: '15px 0'}}>
  <div style={{backgroundColor: '#e6ffed', padding: '10px', borderRadius: '5px'}}>
    打开FCL启动器 → 设置 → Java虚拟机选项 → 更改"-Xmx"参数值
  </div>
</div>

推荐内存分配：
- 基础游戏：1-2GB
- 适中模组：2-3GB
- 大型模组包：3-4GB或更多

### 2. 解决模组冲突

<div style={{border: '1px solid #ddd', borderRadius: '6px', padding: '15px', marginBottom: '20px'}}>
  <p style={{fontWeight: 'bold'}}>步骤：</p>
  <ol style={{marginLeft: '20px'}}>
    <li>暂时移除所有模组</li>
    <li>逐个添加模组，每次添加后测试游戏</li>
    <li>当游戏再次崩溃时，上一个添加的模组可能是冲突源</li>
    <li>验证模组版本与您的Minecraft版本兼容</li>
  </ol>
</div>

### 3. Java版本问题

不同版本的Minecraft需要不同版本的Java：

    </tr>
  </tbody>

## 高级排查方法

对于难以解决的崩溃问题，您可以：

<div style={{backgroundColor: '#eef6fc', padding: '12px', borderRadius: '4px', marginTop: '15px'}}>
  <ul style={{marginLeft: '20px'}}>
    <li>使用FCL内置的"诊断模式"启动游戏</li>
    <li>尝试不同版本的Forge或Fabric加载器</li>
    <li>检查您的设备是否满足最低系统要求</li>
    <li>确保FCL启动器版本是最新的</li>
  </ul>
</div>

如果以上方法都无法解决问题，请将崩溃日志分享到FCL官方社区获取帮助。
